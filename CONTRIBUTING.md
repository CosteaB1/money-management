# Contributing

Thanks for your interest! This is a self-hosted, single-user personal finance app. Bug reports, ideas, and PRs are welcome.

## Getting set up

See the [README](./README.md) for prerequisites and local setup (PostgreSQL, .NET user-secrets, running the API and web app). Architecture notes live in [WIKI.md](./WIKI.md), [BACKEND.md](./BACKEND.md), and [FRONTEND.md](./FRONTEND.md); the manual QA protocol is in [QA.md](./QA.md).

## Branching & pull requests

`main` is always releasable and protected — don't commit to it directly.

1. Branch off `main` with a descriptive prefix: `feat/…`, `fix/…`, `docs/…`, `chore/…`, `test/…`.
2. Make your change with tests, and keep the docs in sync (see below).
3. Open a pull request into `main`. CI (build + backend & frontend test suites) must pass before merge.
4. Keep PRs focused; prefer small, reviewable changes.

## Commit messages

Use [Conventional Commits](https://www.conventionalcommits.org/): `feat:`, `fix:`, `docs:`, `refactor:`, `test:`, `chore:`. Example: `feat(accounts): add account archiving`.

## Tests

- **Backend:** `dotnet test` from the repo root. The Domain and Application suites need no infrastructure; the Infrastructure and Api suites need a running PostgreSQL and a `POSTGRES_PASSWORD` env var (they use a throwaway `money_management_inttest` database and refuse to touch any other).
- **Frontend:** `cd web && npm test` (Vitest).

Please make sure both suites pass before opening a PR.

## Keep the docs in sync

Any change that affects behavior or architecture should update the relevant doc (`WIKI.md` product-level, `BACKEND.md` architecture/data model, `FRONTEND.md` UI/components) in the same PR.

## Never commit secrets or personal data

- Connection strings come from **.NET user-secrets**; the DB password comes from the `POSTGRES_PASSWORD` env var (`.env` is git-ignored — copy from `.env.example`). `.mcp.json` is git-ignored too (copy from `.mcp.json.example`).
- The bank-statement test fixtures are **synthetic** (generated by `tools/MaibFixtureGenerator`). Never add real statements, exports, or account data.
